## 📝 고객 요구사항 정리

- 장바구니에 담긴 상품 갯수 실시간 업데이트
- 배포 URL 제공

- 상품목록 페이지 요구사항

  - 상품 이미지 중심의 레이아웃 구성
  - 장바구니 품목 수 실시간 반영
- 작업 결과물
  - 테스트용 URL 전달
  - 테스트 후 피드백 반영 예정

## 📋 기능 목록 상세 설계

### 1. 상품 목록 조회 및 레이아웃 구성

#### 기능 설명
- 상품 목록 페이지에 상품 리스트 조회 및 카드 형태 노출
- 모바일 퍼스트(최적화), 이후 화면 크기에 따른 반응형 레이아웃 적용

#### 세부 작업
- 상품 데이터 준비 (mock 데이터 또는 API)
- 상품 리스트 렌더링
- Grid 기반 레이아웃 구성
- 반응형 컬럼 수 제어 (모바일: 2열 / 웹: 3~4열)

#### 사용 컴포넌트
- `<ProductList />`

### 2. 상품 카드 UI 구성

#### 기능 설명
- 개별 상품 정보를 카드 형태로 노출
- 상품 이미지, 상품명, 가격 정보 표시

#### 세부 작업
- 상품 이미지 렌더링
- 상품명 및 가격 텍스트 노출
- 카드 영역 UI 구성

#### 사용 컴포넌트
- `<ProductCard />`

### 3. 상품 담기 버튼 및 상태 UI

#### 기능 설명
- 상품 카드 내 장바구니 담기 버튼 제공
- 상품 담김 여부에 따른 버튼 상태 변화 적용

#### 세부 작업
- 담기 버튼 클릭 이벤트 처리
- 상품 담김 상태 관리
- 담김 / 미담김 상태에 따른 버튼 UI 변경

#### 사용 컴포넌트
- `<ProductCard />`
- `<AddToCartButton />`

### 4. 장바구니 상태 관리

#### 기능 설명
- 장바구니 상품 목록 전역 상태 관리

#### 세부 작업
- 장바구니 상태 구조 정의
- 상품 추가 / 제거 로직 처리
- 전역 상태 업데이트 반영

#### 사용 컴포넌트 / 도구
- Recoil *(또는 Context API / Zustand)*
- `<CartStore />`

### 5. 헤더 영역 장바구니 정보 표시

#### 기능 설명
- 헤더 영역에 장바구니 아이콘 및 상품 수 노출
- 장바구니 상태 변경 시 실시간 수량 반영
- 장바구니 상품 수에 따른 UI 상태 분기 적용

#### 세부 작업
- 헤더 UI 구성
- 장바구니 상품 수 계산
- 상태 변경에 따른 렌더링 반영
- 상품 수 0일 경우 뱃지 미노출 또는 비활성화 처리

#### 사용 컴포넌트
- `<Header />`
- `<CartIcon />`
- `<CartCountBadge />`

### 6. 상품 목록 상단 정보 영역

#### 기능 설명
- 상품 목록 페이지 상단 제목 및 부가 설명 노출

#### 세부 작업
- 상품 목록 제목 텍스트 표시
- 전체 상품 개수 계산 및 노출
- 부가 설명 문구 렌더링

#### 사용 컴포넌트
- `<ProductListHeader />`

### 7. (선택) 무한 스크롤 기능

#### 기능 설명
- 스크롤 하단 도달 시 상품 목록 자동 로딩 적용

#### 세부 작업
- 스크롤 위치 감지
- 추가 데이터 로딩 트리거 처리
- 기존 상품 목록에 데이터 병합

#### 사용 컴포넌트 / 기술
- `<InfiniteScroll />`
- Intersection Observer API

### 8. (선택) 로딩 상태 UI

#### 기능 설명
- 상품 목록 데이터 로딩 중 로딩 상태 UI 노출

#### 세부 작업
- 데이터 로딩 상태 관리
- 로딩 중 Skeleton 또는 Spinner 표시
- 로딩 완료 시 상품 목록 노출

#### 사용 컴포넌트
- `<LoadingSpinner />` 또는 `<SkeletonCard />`
