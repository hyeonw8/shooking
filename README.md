# 🛍️ 쇼핑몰 프로젝트 'shooking'

**shooking**은 웹 기반의 패션 쇼핑몰 프로젝트로,  
**모바일 퍼스트 설계와 모바일 환경에 최적화된 구조**를 기준으로 개발됩니다.

현재 1차 단계에서는 상품 이미지 중심의 UI와  
장바구니 상태를 실시간으로 반영하는 **상품 목록 페이지 구현**에 집중합니다.

이후 장바구니, 결제, 쇼핑몰 연동 기능을 단계적으로 확장할 예정입니다.

---

## 📌 프로젝트 개요

- **프로젝트명**: shooking
- **프로젝트 유형**: 웹 기반 패션 쇼핑몰 프로젝트
- **UI 설계 기준**
  - 모바일 퍼스트 설계
  - 모바일 환경 최적화
  - 웹 환경 반응형 확장
- **현재 개발 단계**
  - 1차: 상품 목록 페이지
  - 2차: 장바구니 페이지
  - 3차: 결제 모듈
  - 4차: 전체 쇼핑몰 기능 통합

---

## 🛠 Tech Stack

- **Frontend**: React, JavaScript (또는 TypeScript)
- **State Management**: Recoil / Context API / Zustand
- **Routing**: React Router
- **Testing**: Jest, Cypress
- **UI Development**: Storybook
- **Deployment**: GitHub Pages, Netlify

## ✨ 주요 기능 (1차 개발 범위)
※ 아래 기능은 현재 1차 개발 단계에서 구현을 목표로 합니다.

### 1. 상품 목록 조회
- 상품 리스트 카드 형태 렌더링
- 모바일 퍼스트 기반 레이아웃
- 반응형 UI 적용  
  (모바일 2열 / 웹 3~4열)

### 2. 상품 카드 UI
- 상품 이미지, 상품명, 가격 정보 표시
- 상품 단위 카드 컴포넌트 구성

### 3. 장바구니 담기 기능
- 상품 카드 내 장바구니 담기 버튼 제공
- 담김 여부에 따른 버튼 UI 변경

### 4. 장바구니 상태 관리
- 전역 상태 기반 장바구니 상품 관리
- 상품 추가 / 제거 기능 제공
- 상태 변경 시 UI 즉시 반영

### 5. 헤더 장바구니 정보 표시
- 장바구니 아이콘 및 상품 수 표시
- 상품 수 0일 경우 비활성 또는 미노출 처리

### 6. 상품 목록 상단 정보 영역
- 페이지 제목 표시
- 전체 상품 개수 노출
- 부가 설명 텍스트 제공

### 7. (선택) 무한 스크롤
- 스크롤 하단 도달 시 상품 목록 추가 로딩
- Intersection Observer API 활용

### 8. (선택) 로딩 상태 UI
- 데이터 로딩 중 Skeleton 또는 Spinner 표시
- 로딩 완료 시 상품 목록 렌더링
